# MMCのアイコンを変更するだけ

テストプレイ用と本番用でアイコンをパッと変えたい時とか。

## memo

MMCのアイコンはなんかちゃんと書き出しても適用されないことがある。キレそう。
どうもPrism LauncherとMMCで挙動が違うらしい。

Prismはverによっても挙動が違うかもしれない。

1. config.cfg からiconKey指定を呼び出し
2. ルートディレクトリからiconKeyに合致するファイルを検索
3. なければゲームディレクトリのicon.pngをアイコンとして採用
4. 名前をiconKeyでアイコンファイルをランチャー内に保存
5. 既に同名のアイコンファイルがあった場合
    1. MMC. ランチャー内のアイコンファイルを上書き
    2. Prism. 上書きせずランチャー内のアイコンファイルを採用

例えば、テスト用のtest_icon.pngをよく使うと思う。
これと同名で肉球の画像を用意し、起動構成に設定した場合、MMCではランチャー内のすべてのテスト用の起動構成のアイコンが肉球に上書きされてしまう。
意味わからん。

逆にPrismではランチャー内の画像が優先されるので、意図した見た目にならないことがある。
これも意味わからん。

アイコンを意図した画像にかつ、ランチャー内の同名のアイコンを変更しないようにしようと思うとアイコンにUUIDみたいな名前を付ける必要がある。
ただこれをすると、ランチャー内に起動構成数分のアイコンが残り続けてしまう。

つみ。
